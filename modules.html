Matching rules

Per source-pair cascade: exact → tolerance → fuzzy → break · every match explainable

PSP ↔ CRM ↔ cTrader · funding

the matching cascade · thresholds set with you
1 · exact — match on psp_ref = crm_ref, amount, currency
2 · tolerance — amount ±€0.50 (fees), value-date window ±2 days (T+2 settlement)
3 · fx — convert at the value-date reference rate, tolerance ±0.15%
4 · fuzzy — beneficiary + amount + 24h window (probabilistic, ≥0.85)
5 · residue → break, typed + queued

Learning matcher

human-approved patterns → new rules
Every manual match an analyst makes is proposed back as a governed, human-approved rule — so the auto-match rate rises over time and the breaks queue shrinks. (The system learns your patterns; no performance numbers are claimed before it has run on your data.)
